# 用 Markdown 写一篇个人技术札记

我想把博客写成一种更轻的记录方式：不一定每篇都要像完整教程，也不一定要把所有结论提前整理得特别漂亮。它更像一个持续更新的工作台，记录一次视觉实验、一次动画判断，或者一个当时还没有完全想明白的小问题。

这篇文章是一个中文示例，用来测试当前博客系统对中文排版、段落节奏和目录结构的支持。

## 为什么用 Markdown

Markdown 的好处是写作成本低。文章内容可以直接放在项目里，跟组件、shader、设计系统一起管理，不需要先搭一个复杂后台。

- 写文章时不用打开管理页面。
- 文件可以跟着 Git 一起版本管理。
- 改标题、摘要、标签都很直接。
- 以后要接 CMS 或数据库，也可以再迁移。

## 我希望博客长什么样

它不应该只是普通卡片列表。这个项目的首页已经有等高线背景、主题切换、loading 动画和音乐控件，所以博客区也应该像这个视觉系统自然延伸出来的一层内容。

分类目录负责秩序，文章标题负责入口，最近更新负责提醒。背景保持运行，是为了让从首页到文章详情的切换更像同一个空间里的视角移动，而不是跳去另一个页面。

## 下一步可以细化什么

现在这版已经可以用 Markdown 写文章，后面可以继续加一些更适合个人博客的功能：

- 中文正文的行高、字距和段落宽度继续精修。
- 给文章加封面图或局部视觉注释。
- 给不同分类设计不同的 hover 动效。
- 给文章详情页增加上一页、下一页导航。
- 做一个只读的全文搜索。

## 小结

先把写作链路跑通，比一开始就做完整后台更重要。你现在只需要新增一个 `.md` 文件，再在目录索引里补一条文章信息，就能让首页目录和文章详情页一起生效。
